# Decision Research Agent v0.1.6

Release preparation date: 2026-07-24.

## Supported Surface

v0.1.6 adds a bounded live producer evaluation contract, the official
LangChain DeepSeek provider protocol, deterministic artifact completion and
operator diagnostics, and stricter generic Evidence capture and finalization.
Required verification remains provider-free. Live execution remains a
separately authorized operation rather than release or CI authority.

The application database remains authoritative for research runs, Evidence,
delivery state, review workflow, leases, and artifact metadata. LangGraph
checkpoints remain execution-position state, and LangSmith remains optional
privacy-first diagnostic tracing rather than business authority.

## Changes

### Bounded Live Producer Evaluation And Required Gate

- Added a deterministic bounded live producer evaluation with a provider-free
  required gate and a separately authorized `observe-live` path.
- Added a tracked-archive Docker lifecycle for protected create,
  application-owned persistence, backend restart, same-key replay, privilege
  inspection, and exact task-owned cleanup.
- Moved the secure-runtime precheck into the exact locked backend image before
  service or provider startup. The public truth and proof taxonomy now
  separates deterministic contract proof, required container authority,
  Historical Reviewed Record, and separately authorized live observation.
- Added CI portability repairs for locked-image precheck execution, timeout
  Evidence persistence, and portable Docker fixture paths.

### Official DeepSeek Provider Protocol

- Routed `deepseek-*` primary and fallback models through the official
  `langchain-deepseek` integration and `ChatDeepSeek`.
- Preserved exact non-empty `reasoning_content` across thinking-mode tool-call
  turns, canonicalized thinking mode, omitted automatic thinking-mode
  `tool_choice`, and bound the documented request timeout.
- Only `deepseek-v4-pro` and `deepseek-v4-flash` are durable supported model
  IDs. `deepseek-chat` and `deepseek-reasoner` retain local fixed-mode
  compatibility semantics only through 2026-07-24 15:59 UTC; this is not a
  post-retirement provider-availability claim.
- Preserved existing OpenAI-compatible non-DeepSeek behavior, credential
  aliases, call budgets, application-owned Evidence, and delivery authority.
  Required verification performs no provider request.

### Frontend And CI Maintenance

- Updated the `actions/setup-node` full commit SHA pin from `6.4.0` to `7.0.0`.
- Updated `Vite` from `8.1.4` to `8.1.5` and locked `TypeScript` from `6.0.3`
  to `7.0.2`.
- These changes affect the contributor/CI toolchain only and introduce no
  runtime API or business-authority change.

### Artifact, Diagnostics, Runtime Caps, And Evidence

- Enforced canonical artifact completion before delivery and added bounded
  operator diagnostics for result, failure, and limiter outcomes without
  changing public errors or business authority.
- Bound the generic researcher and network-search runtime caps with existing
  fail-closed or bounded-continue semantics.
- Added nested Evidence capture from completed source-tool calls with
  deterministic deduplication.
- Closed Evidence finalization around one source-response observation time,
  canonical-artifact citation recomputation, and preservation of frozen
  failure, cancellation, timeout, and Talent Evidence.
- Added canonical public-HTTPS source admission and required-domain request
  alignment while keeping receipt and consumer validation independent and
  fail closed.

### Reviewed Observation And Public Claims

- Retained one bounded DeepSeek observation as a Historical Reviewed Record.
  It is not required CI and is not a current release baseline.
- The reviewed record does not verify source truth, research quality, provider
  quality, provider billing, downstream acceptance, exactly-once execution,
  production readiness, or an SLA.

## Compatibility And Migration

No API schema, database schema, migration, Evidence schema, runtime-authority,
Docker, or Compose change is introduced by this release preparation. It
changes only release identity and documentation contracts.

v0.1.6 includes the already-merged official `langchain-deepseek` dependency
and the existing post-v0.1.5 frontend lock updates. Release preparation does
not change either dependency graph; it updates only the frontend package and
lockfile root version identity.

No caller migration is required for the bounded producer, diagnostic receipt,
artifact-completion, or Evidence closures. The live observation path remains
opt-in and separately authorized. DeepSeek provider configuration should use
the durable supported model IDs; credential aliases remain compatible.

## Rollback

Stop local services before changing application revisions. Rolling back to
v0.1.5 requires reinstalling the v0.1.5 Python and frontend locks so the
official provider integration and frontend toolchain match that revision.

No database migration, Evidence schema conversion, Docker volume conversion,
or API payload rewrite is required by this release preparation. Preserve the
application database, checkpoint database, output storage, and task-owned
Evidence records together during any independent recovery.
